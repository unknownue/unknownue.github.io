## 描述翻译

### 目标

`bevy_image` 缺少文档（相关: #3492）

### 解决方案

移除 `#![expect(missing_docs)]` 并添加文档直到 clippy 满意。

我也移除了 `SAMPLER_ASSET_INDEX` 和 `TEXTURE_ASSET_INDEX` 常量，因为它们缺少文档，并且在整个代码库中似乎没有使用（除非 vscode 和 github 搜索都让我失望了）。

## 这个 Pull Request 的故事

这个 PR 的主要目标是改进 `bevy_image` 模块的文档覆盖度。从技术角度看，这属于代码维护工作，重点是提高代码的可读性和可维护性，而不是添加新功能或修复错误。

### 问题和背景

在 Bevy 游戏引擎中，`bevy_image` 模块负责处理图像资产，包括加载、格式转换和纹理处理。在 PR 创建时，该模块缺少大量的文档注释，这违反了 Rust 的最佳实践，并且对于新的贡献者和用户来说，理解和使用这些 API 变得困难。

模块顶部的 `#![expect(missing_docs)]` 属性表明开发者知道文档缺失问题，但暂时抑制了相关的编译器警告。这种状态不应持续太久，因为它会隐藏真实的问题并降低代码质量。

### 解决方案方法

作者采取了直接的方法来解决这个问题：移除 `expect` 属性，然后系统地为所有公共 API 添加文档注释。这种方法强制编译器重新启用缺失文档的警告，确保所有公共项都有相应的文档。

此外，作者还识别并移除了两个未使用的常量（`SAMPLER_ASSET_INDEX` 和 `TEXTURE_ASSET_INDEX`），这是一个典型的代码清理步骤，有助于减少技术债务。

### 实现细节

实现过程涉及为多种类型的项目添加文档：
1. **结构体和枚举**：为 `CompressedImageSaver`、`ExrTextureLoaderSettings`、`HdrTextureLoaderSettings` 等添加文档
2. **错误类型**：详细描述每种错误的可能原因和上下文
3. **函数和方法**：解释参数、返回值和可能出现的错误
4. **常量和字段**：说明它们的用途和预期值

一个重要的技术细节是在多个地方将 `format.pixel_size()?` 替换为 `format.pixel_size().unwrap()`。这种变化表明：
- 对于特定格式（如 `Rgba32Float`），像素大小是确定性的
- 使用 `unwrap()` 避免了不必要的错误传播，简化了错误处理
- 添加了断言来验证假设（如 `debug_assert_eq!(pixel_size, 4 * 4, "Format should have 32bit x 4 size")`）

### 技术洞察

这个 PR 展示了 Rust 文档系统的最佳实践：
1. **文档注释格式**：使用标准的 `///` 注释和 Markdown 格式
2. **错误文档**：使用 `# Errors` 部分详细说明可能出现的错误情况
3. **示例用法**：虽然没有添加代码示例，但文档清楚地解释了每个 API 的用途

移除未使用的常量是一个好的实践，因为它：
- 减少了代码的认知负担
- 消除了潜在的混淆源
- 简化了未来的重构工作

### 影响

这个 PR 的主要影响是：
1. **改善开发者体验**：现在开发者可以通过文档更好地理解 `bevy_image` 模块的 API
2. **启用编译器检查**：移除了 `expect` 属性后，未来的文档缺失将再次触发编译器警告
3. **代码清理**：移除未使用的常量减少了代码库的复杂度
4. **遵循 Rust 惯例**：符合 Rust 社区对文档的高标准要求

### 3. `crates/bevy_image/src/exr_texture_loader.rs`
**变化**: 改进错误处理和文档
**原因**: 简化错误类型，提供更清晰的错误信息

```rust
// 之前:
#[derive(Debug, Error, TypePath)]
pub enum ExrTextureLoaderError {
    #[error(transparent)]
    Io(#[from] std::io::Error),
    #[error(transparent)]
    ImageError(#[from] image::ImageError),
    #[error("Texture access error: {0}")]
    TextureAccess(#[from] TextureAccessError),
}

// 之后:
#[derive(Debug, Error, TypePath)]
pub enum ExrTextureLoaderError {
    /// I/O Error.
    #[error(transparent)]
    Io(#[from] std::io::Error),
    /// Failed to decode the texture.
    #[error(transparent)]
    ImageError(#[from] image::ImageError),
    // 移除了 TextureAccess 错误变体
}
```

### 4. `crates/bevy_image/src/hdr_texture_loader.rs`
**变化**: 类似的错误处理和文档改进
**原因**: 保持一致性，提高代码质量

```rust
// 错误处理改进示例:
// 之前:

// 之后:
// `Rgba32Float` will always return a valid pixel size
```
