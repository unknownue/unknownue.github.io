## Description Translation

### 目标
`scene_viewer` 示例无法正确地在动画之间进行切换。它会播放新的动画，但同时也会继续播放之前的所有动画，导致动画图将它们全部混合在一起。

我怀疑这是在 #11989 中引入动画图时意外破坏的，但我没有确认。

### 解决方案
停止所有动画，然后播放新的动画。

### 测试
```sh
# 按回车键切换动画，或按空格键暂停。
cargo run --example scene_viewer --features "free_camera" -- "assets/models/animated/Fox.glb"
```

## The Story of This Pull Request

这次修复针对的是一个具体的用户体验问题：在 `scene_viewer` 工具中，按回车键切换模型动画时，动画会“堆叠”在一起，而不是干净地切换到新的动画。这对于一个用来查看和调试资产的基本工具来说，是一个严重的功能缺陷。

**问题根源在于 `AnimationPlayer` 的状态管理**。在 PR 描述中，作者推测问题可能是在 #11989 引入动画图系统时引入的。观察修复前的代码逻辑可以印证这一点。当按下回车键时，代码的逻辑是：1) 记录当前是否所有动画都处于暂停状态 (`resume`)，2) 对所有动画执行 `rewind_all().pause_all()`，3) 切换到下一个动画片段，4) 播放新动画，5) 如果之前不是暂停状态 (`resume` 为 `true`)，则 `resume_all()`。

问题出在 `pause_all()` 这个方法上。`pause_all()` 只是暂停动画的*播放进度*，并不会将动画从 `AnimationPlayer` 的活动状态中移除。这意味着，之前播放的动画实体（`AnimationGraph` 中的节点）仍然存在，动画系统在每一帧仍然会计算它们的变换并混合到目标骨骼上。当新动画开始播放时，它就与这些被暂停但未停止的旧动画产生了叠加，造成了错误的视觉效果。

**解决方案直接且有效：使用 `stop_all()` 替换 `pause_all()`**。`stop_all()` 的作用是停止动画播放，并将其标记为“完成”，动画系统之后就不会再处理它们。这确保了动画状态机能够干净地重置，从而只让新动画生效。

代码层面的调整也很清晰。首先，变量名从 `resume` 改为 `paused`，使其语义更明确（表示“之前是否已暂停”）。然后，逻辑流程变为：1) 记录当前是否所有动画都处于暂停状态 (`paused`)，2) 调用 `player.stop_all()` 彻底停止所有现有动画，3) 切换到下一个动画片段并开始播放，4) 如果之前就是暂停状态 (`paused` 为 `true`)，则对新动画调用 `pause_all()`。这个流程保证了状态的正确继承（切换后维持暂停/播放状态），同时彻底解决了动画残留混合的问题。

这是一个典型的“状态泄漏”错误。旧的逻辑试图通过暂停来重置状态，但动画系统更深层的设计（动画图）要求显式地停止动画来清理内部节点。这个修复虽然只修改了寥寥数行，但准确地纠正了对 API 的误用，恢复了工具的基本功能。它提醒我们，在使用状态丰富的系统（如动画系统）时，必须清晰理解“暂停”、“停止”、“重播”等操作在框架内部的精确语义，尤其是当底层架构（如引入动画图）发生变更后。

## Key Files Changed

**`examples/tools/scene_viewer/animation_plugin.rs` (+4/-5)**
这是 `scene_viewer` 示例中处理动画控制输入的插件模块。所有修改都集中在 `handle_inputs` 函数内，用于修复按回车键切换动画时的错误行为。

**修改详情:**
```rust
// 文件：examples/tools/scene_viewer/animation_plugin.rs
// 修改前的代码片段（有Bug）：
            let resume = !player.all_paused();
            // set the current animation to its start and pause it to reset to its starting state
            player.rewind_all().pause_all();

            clips.advance_to_next();
            let current_clip = clips.current();
            player.play(current_clip).repeat();
            if resume {
                player.resume_all();
            }

// 修改后的代码片段（已修复）：
            let paused = player.all_paused();
            player.stop_all();

            clips.advance_to_next();
            let current_clip = clips.current();
            player.play(current_clip).repeat();
            if paused {
                player.pause_all();
            }
```
**关键变化分析:**
1.  **变量语义**: `resume` 改为 `paused`，直接记录“是否已暂停”的状态，逻辑更清晰。
2.  **核心修复**: 将 `player.rewind_all().pause_all()` 替换为 `player.stop_all()`。这是修复的关键，`stop_all()` 会停止并清理动画图中的旧动画节点，防止其继续参与混合计算。
3.  **状态恢复逻辑**: 将 `if resume { player.resume_all() }` 改为 `if paused { player.pause_all() }`。因为 `stop_all()` 后所有动画都停止了，所以需要根据之前记录的状态，决定是否将*新播放的*动画也置为暂停。
