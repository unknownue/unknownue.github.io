## The Story of This Pull Request

This PR addresses a documentation issue in the Bevy game engine. The problem originated from a rustc bug that caused Internal Compiler Errors (ICEs) when attempting to generate documentation with example scraping enabled. To work around this issue, Bevy's maintainers had disabled example scraping for numerous examples by setting `doc-scrape-examples = false` in the Cargo.toml file. This was a temporary measure to allow documentation builds to succeed despite the compiler bug.

The solution approach was straightforward: once the rustc bug was resolved, the workarounds could be removed. The implementation involved systematically changing 337 instances of `doc-scrape-examples = false` to `doc-scrape-examples = true` in the Cargo.toml file. Each change restored the ability to include example code in the generated documentation, which is important for users to understand how to use various Bevy features.

From a technical perspective, the `doc-scrape-examples` setting controls whether Rustdoc includes example code in the documentation. When enabled, it allows documentation to show actual usage examples from the codebase, which is particularly valuable for a complex game engine like Bevy. The rustc bug that necessitated the workaround was related to the documentation generation process, and its resolution allowed Bevy to fully leverage Rust's documentation capabilities.

The impact of this change is significant for Bevy's documentation quality. With example scraping re-enabled, users will now see comprehensive examples alongside API documentation, making it easier to understand how to use various components, systems, and features. This improves the developer experience by providing more contextual examples directly in the documentation.

The changes affect examples across all categories including 2D rendering, 3D rendering, ECS (entity component system), UI systems, input handling, audio, and various tools and utilities. Each example now contributes to the documentation, providing a more complete picture of Bevy's capabilities.

## Key Files Changed

The only file modified in this PR is `Cargo.toml`:

### File: Cargo.toml
The changes in this file involve updating the `doc-scrape-examples` setting for 337 examples from `false` to `true`. This restores example scraping functionality that was previously disabled due to a rustc bug.

The pattern of changes is consistent throughout the file. For each example, the following change was made:

```toml
# Before:
[[example]]
name = "context_menu"
path = "examples/usage/context_menu.rs"
# Causes an ICE on docs.rs
doc-scrape-examples = false

# After:
[[example]]
name = "context_menu"
path = "examples/usage/context_menu.rs"
doc-scrape-examples = true
```

The same pattern was applied to examples across all categories, including:
- 2D rendering examples (like `2d_shapes`, `sprite`, `text2d`)
- 3D rendering examples (like `3d_scene`, `pbr`, `lighting`)
- ECS examples (like `ecs_guide`, `hierarchy`, `parallel_query`)
- UI examples (like `button`, `text`, `flex_layout`)
- Input handling examples (like `keyboard_input`, `mouse_input`)
- And many more specialized examples

These changes ensure that all examples will be included in the generated documentation, providing comprehensive usage examples for Bevy's API.
