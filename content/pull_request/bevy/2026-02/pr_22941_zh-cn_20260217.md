# The Story of This Pull Request

这个 Pull Request 解决了一个 API 完整性的问题。在 Bevy 的 ECS（Entity Component System）架构中，变更检测（change detection）是一个核心机制，它依赖于 `Tick` 计数器来追踪系统和组件状态的变化。标准 `World` 提供了一个 `change_tick()` 方法来访问当前的变更标记（tick），这对于编写依赖于变更状态的逻辑至关重要。

然而，`DeferredWorld` 是一个特殊的世界（World）包装器，用于在某些需要延迟执行或缓冲操作的上下文中安全地进行数据访问。在本次 PR 之前，`DeferredWorld` 缺少一个对应的 `change_tick` 方法。这意味着，当开发者在一个需要或正在使用 `DeferredWorld` 的上下文中（例如在触发器（Trigger）或观察者（Observer）内部）编写代码时，他们无法访问当前的变更标记。这破坏了一致性，并可能导致开发者需要绕过 `DeferredWorld` 的抽象来获取这个基础信息，或者无法实现某些依赖变更检测的逻辑。

这个问题的根源是 `DeferredWorld` 的 API 没有完全映射其底层 `World` 的所有必要功能。`change_tick` 是一个基础访问器，其缺失使得 `DeferredWorld` 在某些场景下功能不完整。Issue #22788 很可能报告了在 `DeferredWorld` 上下文中无法进行变更检测相关操作的具体问题。

解决方案非常直接且符合最小化修改原则：在 `DeferredWorld` 上添加一个 `change_tick` 方法。这个方法的实现是简单的委托（delegation）：它调用其内部持有的 `UnsafeWorldCell`（通过 `self.world` 访问）的 `change_tick()` 方法，并将结果返回。

让我们看一下具体的实现。在 `deferred_world.rs` 文件的 `impl<'w> DeferredWorld<'w>` 块末尾，添加了以下方法：

```rust
/// Gets the current change tick of [`DeferredWorld`].
#[inline]
pub fn change_tick(&mut self) -> Tick {
    self.world.change_tick()
}
```

这个方法被标记为 `#[inline]`，这是一个合理的优化选择，因为它是一个简单的 getter 方法，内联可以消除函数调用的开销。它接收一个 `&mut self` 参数，这与底层 `UnsafeWorldCell::change_tick` 的要求一致。返回类型是 `Tick`，即变更标记的类型。

这个变更虽然代码量很小（仅添加了7行，删除了可能无关的1行），但其意义在于完善了 API 的对称性。现在，`DeferredWorld` 提供了与其底层世界相同的变更检测基础能力。这使得在 Bevy 更复杂的执行上下文（如观察者模式）中编写健壮和一致的代码变得更加容易。开发者无需再担心因为使用 `DeferredWorld` 而丢失了变更追踪的能力。

从工程角度看，这是一个典型的“填补空白”式的重构。它不引入新功能，但修复了 API 设计中的一个疏漏，提升了整个代码库的内聚性和开发者体验。这种类型的修改对于维护一个大型、复杂的游戏引擎框架的 API 一致性非常重要。

## Key Files Changed

- `crates/bevy_ecs/src/world/deferred_world.rs` (+7/-1)

**变更描述与原因**：
此文件中的 `DeferredWorld` 结构体缺少 `change_tick` 方法，导致在延迟世界上下文中无法访问当前变更标记。本次修改添加了此方法，完善了 API。

**关键代码修改**：
```rust
// 文件: crates/bevy_ecs/src/world/deferred_world.rs
// 修改位置：在 impl<'w> DeferredWorld<'w> 块的末尾
// 新增了以下方法：

    /// Gets the current change tick of [`DeferredWorld`].
    #[inline]
    pub fn change_tick(&mut self) -> Tick {
        self.world.change_tick()
    }
```
此代码新增了一个公共方法 `change_tick`，它通过调用内部 `world` 字段（类型为 `UnsafeWorldCell`）的同名方法，来获取并返回当前的 `Tick`。
