## 关键文件更改

**文件**: `.github/workflows/security-static-analysis.yml`

这个文件定义了 Bevy 项目的安全静态分析工作流，会在代码推送或拉取请求时自动运行 CodeQL 分析。PR 更新了其中两个步骤使用的 `github/codeql-action` 版本。

**变更 1**: 初始化 CodeQL 步骤
```yaml
# 之前:
- name: Initialize CodeQL
  uses: github/codeql-action/init@cdefb33c0f6224e58673d9004f47f7cb3e328b89 # v4.31.10

# 之后:
- name: Initialize CodeQL
  uses: github/codeql-action/init@b20883b0cd1f46c72ae0ba6d1090936928f9fa30 # v4.32.0
```

这个步骤负责初始化 CodeQL 分析环境，包括设置编程语言和配置分析参数。

**变更 2**: 执行 CodeQL 分析步骤
```yaml
# 之前:
- name: Perform CodeQL Analysis
  uses: github/codeql-action/analyze@cdefb33c0f6224e58673d9004f47f7cb3e328b89 # v4.31.10

# 之后:
- name: Perform CodeQL Analysis
  uses: github/codeql-action/analyze@b20883b0cd1f46c72ae0ba6d1090936928f9fa30 # v4.32.0
```

这个步骤执行实际的分析并上传结果到 GitHub，以便在拉取请求中显示安全警报。

这两个变更将整个 CodeQL 分析流程更新到最新版本，确保使用最新的安全检测能力和错误修复。注释中保留版本号 `v4.32.0` 有助于人工阅读，而提交哈希保证了构建的确定性。
