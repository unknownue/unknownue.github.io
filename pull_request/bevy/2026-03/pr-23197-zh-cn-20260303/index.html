这个Pull Request的故事

这个PR旨在解决一个在Bevy的ECS（Entity Component System）中使用中遇到的实际问题：如何灵活地存储并与实体关联逻辑，特别是在需要按需执行（one-shot）而非每帧运行的情况下。开发者需要一个清晰、可复用的模式来演示这一技术。

问题的核心在于，Bevy ECS的标准范式是注册在 App 上的系统（System）按调度计划（Schedule）运行。然而，在某些场景下，比如过场动画、脚本事件或由UI触发的交互，逻辑需要与特定实体绑定，并且仅在特定条件或事件发生时执行一次。虽然Bevy提供了 SystemId 和通过 Commands 运行系统的能力，但如何将这些元素组合成一个清晰、实用的模式，对于新手甚至是有经验的开发者来说，可能并不直观。

该PR采用的解决方案直接而有效：创建一个完整的、文档齐全的示例。它没有修改引擎核心代码，而是通过添加示例代码来教育和引导开发者。这种方法风险低，价值高，因为它直接增强了项目的文档和可发现性。

实现分为三个清晰的步骤。首先，在 Cargo.toml 中注册新示例，确保其可以被编译和运行，并且被包含在文档生成流程中（doc-scrape-examples = true）。其次，更新 examples/README.md，为新示例添加索引条目，使其在列表中对使用者可见。最后，也是最重要的部分，创建示例文件 examples/ecs/callbacks.rs。

示例代码本身就是一个精心设计的教学工具。它定义了一个 Callback 组件，其唯一字段是一个 SystemId<(), ()>。这个设计是关键：它将一个可执行系统的引用（ID）存储为实体的数据。在 setup_callbacks 函数中，我们看到了三种不同类型的系统如何被注册并存储：

  1. 最简单的闭包系统：一个没有参数、不查询任何数据的纯函数。
  2. 普通查询系统：一个可以访问 Query<&Callback> 的系统，演示了回调系统如何感知到其他同类组件的存在。
  3. 独占性系统：一个接收 &mut World 参数的系统，展示了最强大的、可以任意修改World的能力。

run_callbacks 函数则演示了如何触发这些存储的回调。它遍历所有拥有 Callback 组件的实体，并使用 commands.run_system(callback.system_id) 来执行对应的系统。代码注释中还提供了一个重要的设计提示：在实际应用中，你可能会使用观察者（Observer）来检测某个实体事件，并以此作为触发回调的时机，这比在 Update 中每帧遍历所有回调更高效。

从技术角度看，这个示例揭示了Bevy ECS的灵活性和“系统即数据”的理念。SystemId 是一种将系统（逻辑）作为一等公民进行传递和存储的机制。这种模式用一定的运行时开销（需要通过 Commands 进行间接调用）换来了极大的架构灵活性。它非常适合那些逻辑执行频率低、但需要与实体紧密绑定、且逻辑本身多样化的场景。

这个PR的影响是直接提升了Bevy项目自身的质量。它为开发者提供了一个官方、标准的模式来解决“按需执行实体关联逻辑”的问题，减少了社区重复发明轮子和摸索正确用法的时间。通过将其作为一个可运行的示例，它确保了代码的正确性和可测试性，同时也成为了项目文档的一部分。

关键文件更改

  1. examples/ecs/callbacks.rs (+53/-0)

    • 描述与原因：这是本次PR新增的核心示例文件。它完整演示了如何定义回调组件、注册不同类型的系统到命令中、将系统ID存储在实体上，以及如何遍历并执行这些回调。
    • 关键代码：
      // 定义存储系统ID的组件
#[derive(Component)]
struct Callback {
    system_id: SystemId<(), ()>,
}

// 注册一个可以访问World的独占性系统作为回调
let exclusive_callback = Callback {
    system_id: commands.register_system(|world: &mut World| {
        let n_entities = world.entities().len();
        println!("This is the exclusive callback system. There are currently {n_entities} entities in the world.");
    }),
};

// 执行所有找到的回调
fn run_callbacks(mut commands: Commands, query: Query<&Callback>) {
    for callback in query.iter() {
        commands.run_system(callback.system_id);
    }
}
    • 与PR目的的关系：此文件直接实现了PR的目标，即提供一个关于回调与一次性系统的可工作示例。
