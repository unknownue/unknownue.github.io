描述翻译

我做了大量测试，经过最近几轮对高光全局光照（specular GI）的改进，我认为这个功能（抖动）不再值得使用了。关闭它可以减少薄几何体上的光泄漏（light leaks）。

这个拉取请求的故事

这个PR涉及一个非常具体的渲染优化：禁用Solari实时全局光照系统中世界缓存查询的抖动功能。整个修改只改动了一行代码，但这行代码背后反映了渲染技术栈的演进和工程权衡。

问题的背景 Solari是Bevy引擎的实时全局光照系统。在查询世界缓存时，为了提高采样的质量，原始的代码实现了一个抖动机制。这个机制会在世界位置的切平面上添加一个随机的偏移量，其大小与当前缓存单元的尺寸相关。抖动通常用于打破规则的采样模式，避免出现带状瑕疵或摩尔纹。然而，这个技术有一个副作用：当几何体很薄时，抖动可能导致采样点偏移到几何体的背面，从而产生“光泄漏”现象——即本不该被照亮的区域错误地接收到了光照。

解决方案的演进 根据PR描述，作者JMS55经过大量测试后得出结论：在最近的几轮针对高光全局光照的改进之后，抖动带来的好处已经不足以抵消其引入的光泄漏问题。因此，决定默认关闭这个功能。这是一个典型的性能/质量权衡决策。随着底层算法的改进（specular GI的优化），一个旨在解决旧问题（采样瑕疵）的技术，其收益已经下降，而其成本（光泄漏）变得更加突出，因此移除它是合理的。

实现细节 实现非常简单：将WGSL着色器中的一个预处理条件从 #ifndef NO_JITTER_WORLD_CACHE 翻转为 #ifdef JITTER_WORLD_CACHE。这改变了功能的默认行为：

  • 修改前：除非显式定义了 NO_JITTER_WORLD_CACHE，否则默认启用抖动。
  • 修改后：只有显式定义了 JITTER_WORLD_CACHE，才会启用抖动。

这意味着所有使用Solari的项目将默认获得无抖动的世界缓存查询，从而减少薄几何体的光泄漏。如果某些场景或项目仍然需要抖动来改善其他方面的质量，他们可以通过定义 JITTER_WORLD_CACHE 宏来重新启用它。

技术洞察 这个改动体现了渐进式渲染优化的一个常见模式：随着核心算法的成熟和稳定，一些辅助性的、启发式的技术可能会变得多余甚至有害。抖动是一个典型的“后处理”或“增强”技术，它在核心算法存在局限性时很有用。当核心算法（这里是specular GI）得到显著改进后，这种辅助技术的价值就需要重新评估。

从代码维护的角度看，这个改动也很干净。它没有删除任何代码逻辑，只是改变了默认的编译开关。这保留了在必要时重新启用该功能的能力，同时为大多数用户提供了更好的默认体验。

影响 这个改动对最终用户的影响是正面的：减少了薄几何体上的光泄漏，提高了渲染的物理准确性。对于开发者来说，他们需要了解这个默认行为的改变。如果他们的项目之前依赖抖动来掩盖某些瑕疵，他们可能需要检查渲染效果，并在必要时通过定义宏来恢复旧行为。不过，考虑到作者是基于大量测试做出的决定，很可能绝大多数场景都不会有负面影响。

关键文件变更

文件: crates/bevy_solari/src/realtime/world_cache_query.wgsl

这个文件包含了用于查询世界缓存的WGSL着色器代码。主要更改是反转了控制世界位置抖动的预处理条件。

代码变更:

// 修改前（第54行附近）:
#ifndef NO_JITTER_WORLD_CACHE
    let TBN = orthonormalize(world_normal);
    let offset = (rand_vec2f(rng) * 2.0 - 1.0) * cell_size * 0.5;
    world_position += offset.x * TBN[0] + offset.y * TBN[1];
#endif

// 修改后:
#ifdef JITTER_WORLD_CACHE
    let TBN = orthonormalize(world_normal);
    let offset = (rand_vec2f(rng) * 2.0 - 1.0) * cell_size * 0.5;
    world_position += offset.x * TBN[0] + offset.y * TBN[1];
#endif

说明:

  • 抖动代码的逻辑本身没有改变。它仍然在法线确定的切平面上，生成一个在 [-0.5*cell_size, 0.5*cell_size] 范围内的随机偏移，并应用到世界位置上。
  • 唯一的变化是控制这段代码是否被编译的条件。现在，抖动功能默认是禁用的。
  • 这个更改直接服务于PR的目标：减少光泄漏。通过默认禁用抖动，薄几何体上的采样点就不会被随机推到背面，从而避免了错误的光照计算。
