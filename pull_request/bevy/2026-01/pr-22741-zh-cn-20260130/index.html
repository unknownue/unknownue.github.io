v4.32.0

  • 将默认的 CodeQL 捆绑包版本更新至 2.24.0。#3425

  • 在启用了 Actions 调试功能的情况下运行默认设置工作流时，CodeQL Action 现在将使用更唯一的名称来上传来自 Dependabot 身份验证代理的日志作为工作流工件。这确保了构建矩阵中多个任务之间的工件名称不会冲突。#3409

  • 现在包含在 CodeQL Action 版本中的变更日志摘要更短，以避免在 Dependabot PR 中出现重复信息。#3403

关于这个拉取请求的说明

这个 PR 是典型的依赖维护工作。在软件开发中，保持第三方依赖的更新是基础但关键的任务，特别是对于安全扫描工具这类直接影响代码库安全性的工具。

这个 PR 的核心是更新 GitHub Actions 工作流中使用的 CodeQL 分析工具。Dependabot 检测到 github/codeql-action 有可用更新（从 v4.31.10 到 v4.32.0），自动创建了此 PR。变更只涉及单个文件中的两行，将两个步骤中使用的动作版本号更新到最新。

CodeQL 是 GitHub 的语义代码分析引擎，用于自动化安全扫描。v4.32.0 的主要更新是将底层的 CodeQL 捆绑包（bundle）升级到 2.24.0 版本。捆绑包包含了 CodeQL 命令行工具和预编译的查询库。这意味着 Bevy 项目在后续的 CI 运行中，将使用更新的查询来检测安全漏洞和代码质量问题，可能包括对新类型漏洞的检测或现有查询的改进。

除了主要的捆绑包更新，v4.31.11（此次次要升级序列的一部分）还包含了几项实用的改进。其中值得注意的是，改进了在启用调试模式时的多任务工件命名，避免在构建矩阵中发生冲突。这是一个典型的 DevOps 优化，提高了复杂 CI 配置的可靠性。另外添加了自动排除生成文件的实验性支持，虽然默认未启用，但为未来可能的优化铺平了道路，有助于减少误报和加快扫描速度。

从工程实践角度看，这个 PR 展示了自动化依赖管理的价值。Dependabot 自动处理版本检测、PR 创建和兼容性检查，让维护者可以专注于审查变更内容而非手动更新过程。提交信息中的哈希值 cdefb33c0f6224e58673d9004f47f7cb3e328b89 和 b20883b0cd1f46c72ae0ba6d1090936928f9fa30 是 Git 提交哈希的完整形式，用于精确锁定特定版本，这是 GitHub Actions 中确保构建可重现性的推荐做法。

维护者 mockersf 在大约 52 分钟内审查并合并了这个 PR，这个相对较快的处理速度表明团队对安全工具更新的重视，也反映出这是一个低风险、标准化的依赖更新。标签 C-Dependencies 表明项目有良好的分类系统，便于跟踪不同类型的变更。

关键文件更改

文件: .github/workflows/security-static-analysis.yml

这个文件定义了 Bevy 项目的安全静态分析工作流，会在代码推送或拉取请求时自动运行 CodeQL 分析。PR 更新了其中两个步骤使用的 github/codeql-action 版本。

变更 1: 初始化 CodeQL 步骤

# 之前:
- name: Initialize CodeQL
  uses: github/codeql-action/init@cdefb33c0f6224e58673d9004f47f7cb3e328b89 # v4.31.10

# 之后:
- name: Initialize CodeQL
  uses: github/codeql-action/init@b20883b0cd1f46c72ae0ba6d1090936928f9fa30 # v4.32.0

这个步骤负责初始化 CodeQL 分析环境，包括设置编程语言和配置分析参数。

变更 2: 执行 CodeQL 分析步骤

# 之前:
- name: Perform CodeQL Analysis
  uses: github/codeql-action/analyze@cdefb33c0f6224e58673d9004f47f7cb3e328b89 # v4.31.10

# 之后:
- name: Perform CodeQL Analysis
  uses: github/codeql-action/analyze@b20883b0cd1f46c72ae0ba6d1090936928f9fa30 # v4.32.0

这个步骤执行实际的分析并上传结果到 GitHub，以便在拉取请求中显示安全警报。

这两个变更将整个 CodeQL 分析流程更新到最新版本，确保使用最新的安全检测能力和错误修复。注释中保留版本号 v4.32.0 有助于人工阅读，而提交哈希保证了构建的确定性。
